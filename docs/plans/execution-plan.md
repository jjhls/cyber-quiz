## 1. 执行策略

### 1.1 内部等级决策

**等级: L**（设计 + 分阶段实施）

理由：
- 项目规模适中（14 个页面，约 20 个 API 端点）
- 前后端分离，需要分阶段构建
- 依赖关系清晰，适合流水线式开发
- 不需要 XL 多代理并行（规模 < 100 人）

### 1.2 执行阶段划分

```
Phase 0: 项目初始化与基础设施
    ↓
Phase 1: 数据库与后端核心 API
    ↓
Phase 2: 前端基础框架与用户系统
    ↓
Phase 3: 题库管理与竞赛管理
    ↓
Phase 4: 在线答题与自动判分
    ↓
Phase 5: 排行榜与数据统计
    ↓
Phase 6: 错题本与个人中心
    ↓
Phase 7: 后台管理面板
    ↓
Phase 8: 移动端适配与优化
```

## 4. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| Prisma Session Store 兼容性 | 认证无法工作 | 备选方案：自定义 Session Store |
| Ant Design 暗色主题定制 | 样式冲突 | 使用 CSS 变量覆盖 + ConfigProvider |
| ECharts 在 React 中渲染 | 图表不显示 | 使用 echarts-for-react 封装库 |
| 答题计时器精度 | 计时不准 | 使用服务器时间校准 |
| CSV 导入编码问题 | 中文乱码 | 强制 UTF-8 编码处理 |
